Ch6. Structure and Blueprints

In this chapter, we improve the software architecture of our app to make it more efficient and easier to maintain. We restructure the project using Flask Blueprints so routes, templates, and future logic (forms, Python code, and databases) stay modular as complexity grows.

Paper
Carbon App

Structure and Blueprints (Basic)

  • Create dedicated packages with __init__.py and routes.py.
  • Define Blueprint objects and attach view functions with decorators.
  • Register all blueprints in the central application object.
  • Run the app from application.py.

Structure and Blueprints (Applied)

HTML links must reference blueprint endpoints, for example url_for('home.home_home'), url_for('methodology.methodology_home'), and url_for('carbon_app.carbon_app_home').

Weekly challenge

Add a users package with routes for register and login, and add two new routes in carbon_app for new data entry and user data view. Templates are grouped into templates/users and templates/carbon_app.
